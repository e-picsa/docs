# Add Country

## Database updates

### Country code
Add reference to `geo.countries` table using the [ISO 3166](https://en.wikipedia.org/wiki/List_of_ISO_3166_country_codes) for country code.

E.g., if adding the country "Zimbabwe", should use code `ZW`. Use the `local_name` column to add non-english name variant if popular

```sql
INSERT INTO geo.countries (code, name, local_name)
VALUES ('ZW', 'Zimbabwe', null);
```

### Languages
Once the country has been added register any language codes using the [ISO 639 Set 1 standard](https://en.wikipedia.org/wiki/List_of_ISO_639_language_codes)

Include an `en` variant for localised updates to English language variables, e.g. adding `shona` and `ndebele` languages

```sql
INSERT INTO geo.locales (language_code, name, local_name, country_code)
VALUES
  -- Zimbabwe
  ('en', 'English', NULL, 'ZW'),
  ('sn', 'Shona', 'ChiShona', 'ZW'),
  ('nd', 'Ndebele', 'isiNdebele', 'ZW'),
```

Additionally update the `public.translations` table to include the columns

```sql
ALTER table public.translations ADD en-ZW text;
ALTER table public.translations ADD sn-ZW text;
ALTER table public.translations ADD nd-ZW text;
```

## Map and Geo Data

The E-PICSA app includes a static export of map tiles and geojson boundaries of administrative areas for included countries.
This enables users to view interactive maps offline within the climate tool.

A standalone repository has been setup for managing these processes, with highly optimised workflows to keep file size small
https://github.com/e-picsa/geo-data

The interactive dashboard can be accessed from:
https://geo-data-api.picsa.app

1. Export TopoJson and place in `libs/data/geoLocation`

2. Exported map tiles and place in `apps/picsa-tools/climate-tool/src/assets/mapTiles`

3. Manually populate lists of locations to corresponding province/district exports for the country in the 
`libs/data/geolocation` directory. These can be manually extracted from the TopoJson (this should be automated in the future).

4. Update geoLocation index to include data. Correct labels for administrative boundary levels (e.g province/district/) can be found
on the [Open Street Map Wiki](https://wiki.openstreetmap.org/wiki/Tag:boundary%3Dadministrative#Table_:_Admin_level_for_all_countries)

`libs\data\geoLocation\index.ts`
```ts
const GEO_LOCATION_DATA: { [country_code in ICountryCode]?: IGelocationData } = {
  zw: {
    topoJson: async () => {
      const res = await import('./zw/ZW.topo.json');
      return res.default;
    },
    admin_4: {
      label: translateMarker('Province'),
      locations: ZW_PROVINCES,
    },
  },
};
```

## Extension App Theming

Each variation of the extension app is themed to match the country.

Colours are typically chosen to reflect national [flag color codes](https://www.flagcolorcodes.com/),
and generated as a theme using legacy [Material Design Palette Generator](https://moikaturns.github.io/mcg) [GitHub](https://github.com/mbitson/mcg)

`./libs/theme/src/themes/picsa-zw.scss`
```scss
@use 'mixins';
@use '@angular/material' as mat;

$primary: (
  // ...
  500: #009739,
  // ...
)
$secondary: (
  // ...
  500: #fce300,,
  // ...
)

$palette-primary: mat.m2-define-palette($primary);
$palette-accent: mat.m2-define-palette($accent, A200, A100, A400);
$palette-warn: mat.m2-define-palette(mat.$m2-red-palette);

$theme-zw: mat.m2-define-light-theme(
  (
    color: (
      primary: $palette-primary,
      accent: $palette-accent,
      warn: $palette-warn,
    ),
  )
);

body[data-theme='picsa-zw'] {
  @include mat.all-component-themes($theme-zw);
  @include mixins.setColorVariables($palette-primary, $palette-accent, $palette-warn);
}
```
The theme should be exported from the main theming index to be available to the app

`./libs/theme/src/themes/_index.scss`
```scss
@import './picsa-zw.scss';
```
